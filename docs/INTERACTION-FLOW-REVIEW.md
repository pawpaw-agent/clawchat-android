## 2. 发现的问题

### 2.1 🔴 高优先级问题

#### P1-1: 自动连接静默失败

**位置**: `MainViewModel.autoConnectIfNeeded()`

**问题**:
```kotlin
result.onFailure { error ->
    Log.w(TAG, "Auto-connect failed: ${error.message}")
    _uiState.update { it.copy(isLoading = false) }
    // 不显示错误，让用户手动连接  ← 问题所在
}
```

**影响**: 用户打开 App 后看到空白会话列表，不知道发生了什么。

**建议**: 
- 显示连接失败提示
- 提供"重试"按钮
- 显示上次连接的 Gateway 地址

#### P1-2: 会话列表无实时更新

**位置**: `MainViewModel.loadSessionsFromGateway()`

**问题**: 会话列表仅在连接成功时加载一次，后续新会话/消息不会更新列表。

**影响**: 
- 用户在其他设备创建会话，当前设备看不到
- 消息数、最后消息预览不会更新
- 需要手动刷新（但没有刷新按钮）

**建议**:
- 监听 `session.created` / `session.deleted` 事件
- 添加下拉刷新
- 定时轮询更新（作为备选）

#### P1-3: 流式输出体验差

**位置**: `SessionScreen.MessageItem`

**问题**: 
- 流式文本 (`chatStream`) 没有特殊渲染
- 没有打字机效果
- 没有光标指示

**影响**: 用户不知道助手正在输出，体验不如 WebChat。

**建议**:
- 添加流式光标动画
- 实现打字机效果
- 显示"正在输出..."状态

### 2.2 🟡 中优先级问题

#### P2-1: 会话状态显示不清晰

**位置**: `MainScreen.SessionItem`

**问题**: 会话卡片不显示当前状态（运行中/暂停/等待中）。

**影响**: 用户不知道会话是否活跃。

**建议**: 添加状态指示器（如设计概念中的脉冲条）。

#### P2-2: 长按选项过多

**位置**: `MainScreen.SessionOptionsDialog`

**问题**: 长按会话显示 4 个按钮（暂停/恢复、重命名、终止、删除），操作繁琐。

**建议**: 
- 常用操作放在卡片上（如删除）
- 不常用操作收起到菜单
- 添加左滑删除手势

#### P2-3: 消息发送状态缺失

**位置**: `SessionScreen.MessageItem`

**问题**: 用户消息没有发送状态指示（发送中/已发送/失败）。

**建议**: 添加消息状态图标（时钟 → ✓ → ✓✓ 或 ⚠️）。

#### P2-4: 工具调用显示简陋

**位置**: `SessionScreen` 中的 `chatToolMessages`

**问题**: 工具调用只显示名称和输出，没有折叠/展开，没有进度指示。

**建议**: 
- 添加折叠/展开功能
- 显示工具参数
- 显示执行时间

### 2.3 🟢 低优先级问题

#### P3-1: 搜索功能有限

**问题**: 只能搜索会话标题和最后消息，不能搜索消息内容。

**建议**: 后续支持全局消息搜索。

#### P3-2: 消息引用/复制

**问题**: 长按消息只能删除，不能复制或引用。

**建议**: 添加复制、引用、分享功能。

#### P3-3: 没有消息时间戳

**问题**: 消息时间显示为"刚刚/X分钟前"，不够精确。

**建议**: 点击显示详细时间。

## 6. 实现建议

### 6.1 立即可做的快速修复

```kotlin
// 1. 自动连接失败提示
// MainViewModel.autoConnectIfNeeded()
result.onFailure { error ->
    _uiState.update { it.copy(
        isLoading = false,
        connectionError = error.message  // 新增字段
    )}
    _events.trySend(UiEvent.ShowConnectionError(error.message ?: "连接失败"))
}

// 2. 添加下拉刷新
// MainScreen.kt
val refreshState = rememberPullToRefreshState()
PullToRefreshBox(
    state = refreshState,
    onRefresh = { viewModel.refreshSessions() }
) {
    SessionList(...)
}

// 3. 流式光标
@Composable
fun StreamingCursor() {
    val blink by animateFloatAsState(
        targetValue = if (visible) 1f else 0f,
        animationSpec = infiniteRepeatable(
            animation = tween(500),
            repeatMode = RepeatMode.Reverse
        )
    )
    Text("█", color = Color.White.copy(alpha = blink))
}
```

### 6.2 需要架构调整的改进

1. **实时会话更新**: 需要监听 Gateway 事件，更新本地缓存
2. **消息发送状态**: 需要跟踪每条消息的发送状态
3. **工具调用进度**: 需要解析 `phase: "update"` 事件

## 7. 总结

ClawChat Android 当前实现了核心功能，但在实时性和交互体验上与 WebChat 有明显差距。

**最关键的 3 个问题**:
1. 自动连接静默失败，用户无感知
2. 会话列表无实时更新
3. 流式输出体验差

**建议优先修复顺序**:
1. 连接失败提示 + 重试机制
2. 添加下拉刷新
3. 流式光标效果
4. 会话状态指示
5. 工具调用卡片优化
